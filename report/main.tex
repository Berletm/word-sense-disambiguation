\section{Задание.}

\begin{tcolorbox}[title=Условие задания, colframe=blue, fonttitle=\bfseries]
Реализовать систему, которая:

определяет наиболее подходящее значение многозначного слова в контексте (word sense disambiguation, WSD — снятие лексической неоднозначности);
извлекает и выводит семантические отношения (синонимы, гиперонимы, гипонимы) для выбранного значения. 

\emph{Замечание: слово, для которого Вы выводите семантические отношения отмечаем сами. Например: 
Я буду сдавать лабораторные работы вовремя и получу хорошую оценку за экзамен.  
Отмеченное слово “оценка” . Для него решаем WSD задачу.}
\\\\
Для решения задачи необходимо использовать тезаурус WordNet или его аналог (например, RuWordNet, РуТез и др.), а также реализовать алгоритм Леска.
Использование готовых реализаций WSD запрещено.

\section{Выполнение.}
\subsection{Описание алгоритма Леска}
    \begin{itemize}
        \item Вокруг целевого слова берется контекст - его окружение.
        \item Из тезаруса извлекаются всевозможные определения целевого слова.
        \item Вычисляется пересечение слов из определения и контекста.
        \item Выбирается то определение, которое имеет больше всего общих слов с контекстом.
    \end{itemize}
\subsection{Описание изменения в алгоритме}
    В данной реализации было изменено сравнение контекста и определения: вместо простого пересечения вычисляется вектор контекста с помощью предобученной модели и для каждого определения вычисляется вектор.
    Затем между этими векторами считается косинус угла. В качестве итогового берется определение с наименьшим значением.
